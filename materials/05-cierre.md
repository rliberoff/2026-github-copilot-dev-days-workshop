# Cierre y Retrospectiva

**Duración estimada**: 5 minutos
**Objetivo**: Reflexionar sobre lo aprendido, comparar las experiencias greenfield y brownfield, y definir próximos pasos para adoptar SDD en tu equipo.

## Preguntas de reflexión

Toma unos minutos para pensar en estas preguntas:

### Sobre el flujo SDD

1. **¿Qué fase del flujo SDD agregó más claridad a tu trabajo?**
   - ¿Fue la especificación, el plan, las tareas, o la implementación guiada?
   - ¿Por qué esa fase fue la más valiosa?

2. **¿Cómo se resolvieron las ambigüedades durante el proceso?**
   - ¿Usaste `/speckit.clarify`? ¿Qué preguntas reveló?
   - ¿Hubo momentos donde el plan te hizo replantear la especificación?

### Greenfield vs Brownfield

1. **¿Qué diferencias concretas encontraste entre los dos enfoques?**
   - ¿En qué se diferenció el proceso de especificación?
   - ¿Qué riesgos son únicos de cada escenario?
   - ¿Cómo cambió tu estrategia de testing?

2. **¿Cuál de los dos escenarios refleja mejor tu trabajo diario?**
   - ¿Qué adaptaciones necesitarías para aplicar SDD en ese contexto?

### Personalización y adopción

1. **¿Qué personalizaciones necesitaría tu equipo?**
   - ¿Qué secciones agregarías a las plantillas de spec o plan?
   - ¿Qué comandos de verificación serían útiles como extensiones?

## Artefacto de cierre

Completa la [plantilla de retrospectiva](retrospectiva.md) y haz commit en tu repositorio:

```bash
# Edita el archivo con tus reflexiones
code materials/retrospectiva.md

# Commitea como evidencia de aprendizaje
git add materials/retrospectiva.md
git commit -m "docs: complete workshop retrospective"
```

## Próximos pasos

1. **Practica SDD en tu próximo proyecto**: Elige un feature pequeño y aplica el flujo completo
2. **Crea un preset para tu equipo**: Personaliza las plantillas con las convenciones de tu organización
3. **Comparte con tu equipo**: Presenta los conceptos y artefactos generados durante el workshop
4. **Explora los pasos opcionales**: Si no los usaste todos (clarify, checklist, analyze), pruébalos en tu contexto
